1 Presentation
We have to do the work schedule for a repair shop under constraints listed below:
We have only 3 employees.
Its a 8 hours working day.
There are 12 orders on this day.
Orders are allowed to be done the next day but it has a cost.
To start xing a device, the employee needs to receive spare parts for it.
Each repair task has a specic duration.
Each order have to be delivered before the end of working hours, we know
duration of delivery.
Its possible for an employee to work during the delivery of spare pieces
and the delivery of xed orders.
Our goal is to minimize the amount of penalty therefore we have to nd the
best allocations of employees working time in the best order to proceed.

3 Results:
With these data given by the solver, we nd an objective value of 240 euros.
We can see that only 3 orders are reported to the next day.
The column family is an analysis of the results that we have, each employee
can take a number, except 0 to organize his daily work. The main interest of
that column was to help the search of symmetries.
3.1 Symmetries :
We can see that data are really dierent from one to another, we dont have
2 data with the same set of parameters. As decision variables are using a lot
of data parameters we can believe that the solution is close to be unique. An
interesting observation was that the solver is always giving the same solution.
We tried to permute the order with the lowest start and the order with the
highest one of each Family but it didnt give any solution, we tried a lot of
permutations but it never gave any solution. We truly believe that the model use
good decision variables. For example a model that try to create an allocation of
an employee to a task during a limited time will have dierent decision variables
that should create symmetries.
